# Errors Reference

在需要解释 CLI 返回值、判断是否自动重试或定位错误时读取本文档。

## 响应格式

```json
{"c":0,"d":{...}}
{"c":0}
{"c":40001,"m":"..."}
```

- `c = 0`：成功
- `d`：成功时的数据
- `m`：失败时的错误信息

## 错误分类

- `400xx`：参数或状态错误
- `401xx`：认证问题
- `403xx`：权限不足
- `429xx`：限流
- `500xx`：服务端错误

## 优先处理规则

- `40101`：重新登录
- `40102`：先 `client auth refresh`，再重试
- `40103`：确认后重置 API Key
- `40001`：检查参数
- `40002`：检查 UID 或资源是否存在
- `40004`：当前状态不允许该操作，例如非好友发消息
- `42901` / `42902`：等待后重试
- `50001` / `50002` / `50003`：指数退避重试，最多 3 次

## 重试策略

- 第 1 次：等待 1 秒
- 第 2 次：等待 2 秒
- 第 3 次：等待 4 秒
- 超过 3 次仍失败：停止并向用户说明

## 自动处理边界

- 可以自动处理：`40102`、部分 `429xx`、部分 `500xx`
- 不要自动处理：`40101`、`40103`、涉及用户确认的破坏性或外发动作

## 对用户的反馈方式

- 说明错误码、错误含义、已执行的恢复动作、下一步建议。
- 如果已经自动重试过，要明确告诉用户重试次数和结果。
