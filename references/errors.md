# 错误码 - 详细文档

> 本文档按需加载，命令中无需包含路径前缀。直接使用 `client` 命令（Windows 系统使用 `client.exe`）。

## 响应格式

所有命令返回 JSON 到 stdout：

```json
{"c": 0, "d": { ... }}    // 成功，有数据
{"c": 0}                   // 成功，无数据
{"c": 40001, "m": "..."}  // 错误，m 为错误描述
```

- `c`: 错误码，`0` 表示成功
- `d`: 响应数据（成功时）
- `m`: 错误描述（失败时）

## 错误码分类

| 前缀 | 类别 | 说明 |
|------|------|------|
| `0` | 成功 | 正常响应 |
| `400xx` | 客户端错误 | 请求参数错误、资源不存在等 |
| `401xx` | 认证错误 | 未登录、Token 过期、API Key 失效 |
| `403xx` | 权限错误 | 权限不足 |
| `429xx` | 限流 | 请求频率超限 |
| `500xx` | 服务端错误 | 后端服务异常 |

## 完整错误码表

### 认证相关 (401xx)

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `40101` | 未登录 | 运行 `client auth login <手机号> <验证码>` |
| `40102` | JWT 过期 | 运行 `client auth refresh` 自动刷新 |
| `40103` | API Key 失效 | 运行 `client auth key --regenerate` 重新生成 |
| `40104` | 认证信息格式错误 | 检查凭证文件是否损坏，重新登录 |

### 权限相关 (403xx)

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `40301` | 权限不足 | 当前操作需要更高权限，或操作对象不属于你 |
| `40302` | 账号未激活 | 检查账号状态 |

### 客户端错误 (400xx)

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `40001` | 请求参数错误 | 检查请求参数格式和必填字段 |
| `40002` | 资源不存在 | 检查 UID 是否正确 |
| `40003` | 资源已存在 | 如重复注册、重复发送好友请求等 |
| `40004` | 操作不允许 | 当前状态不允许此操作（如对非好友发送消息、重复接受已处理的请求） |
| `40005` | 内容格式错误 | 检查字段长度、格式等约束 |

### 限流 (429xx)

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `42901` | 请求频率超限 | 等待后重试，建议间隔 1-2 秒 |
| `42902` | 验证码发送频率超限 | 等待冷却期后重试（通常 60 秒） |

### 服务端错误 (500xx)

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `50001` | 内部服务错误 | 指数退避重试（1s→2s→4s），最多 3 次 |
| `50002` | 数据库错误 | 稍后重试 |
| `50003` | 第三方服务错误 | 稍后重试，持续失败则联系管理员 |

### CLI 专用错误

| 错误信息 | 说明 | 处理方式 |
|----------|------|----------|
| `not logged in` | 未登录 | 运行 `client auth login` |
| `command not found` | 命令不存在 | 运行 `client --help` 查看可用命令 |
| `invalid argument` | 参数无效 | 运行对应命令的 `--help` |

## 错误处理策略

### 自动恢复（无需人工干预）

以下错误可以自动处理，不需要通知用户：

| 错误码 | 自动处理 |
|--------|----------|
| `40102` | 自动运行 `auth refresh`，然后重试原命令 |
| `42901` | 等待 2 秒后自动重试，最多 3 次 |
| `42902` | 告知用户需要等待 60 秒后重试 |
| `50001` | 指数退避重试（1s→2s→4s），最多 3 次 |
| `50002` | 稍后重试 |
| `50003` | 稍后重试，持续失败则通知用户 |

### 需要人工干预

以下错误需要用户介入：

| 错误码 | 处理方式 |
|--------|----------|
| `40101` | 需要用户重新提供手机号和验证码登录 |
| `40103` | 通知用户 API Key 失效，确认后运行 `auth key --regenerate` |
| `40301` | 告知用户权限不足，说明当前操作的约束 |
| `40001` | 检查参数后告知用户具体问题 |
| `40002` | 告知用户资源不存在，检查 UID |
| `40003` | 告知用户资源已存在（如已发送过好友请求） |
| `40004` | 告知用户操作不允许（如对非好友发消息） |

### 重试策略

对于可重试的错误（限流、服务端错误），使用以下策略：

```
第 1 次重试：等待 1 秒
第 2 次重试：等待 2 秒
第 3 次重试：等待 4 秒
超过 3 次：放弃，通知用户
```

## 常见问题

**Q: 所有命令都返回 40101 怎么办？**
A: 凭证可能已失效。运行 `client auth refresh` 刷新。仍然失败则需要重新登录。

**Q: 频繁收到 42901 限流怎么办？**
A: 降低请求频率，命令之间增加间隔（至少 1 秒）。

**Q: 500xx 错误持续出现？**
A: 可能是后端服务异常，等待一段时间后重试。如果持续超过 10 分钟，可能是服务故障。

**Q: 40103 API Key 失效怎么处理？**
A: 运行 `client auth key --regenerate` 重新生成。注意旧 Key 会立即失效，需要重启 WebSocket 服务。
