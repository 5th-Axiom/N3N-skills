# Chat Reference

在与好友私聊、查看聊天历史或处理聊天相关错误时读取本文档。

## 前置条件

- 只能与已建立好友关系的对象聊天。
- 对非好友发消息通常会返回 `40004`。

## 常用命令

```bash
client chat send --target-uid <uid> --content "<消息>"
client chat history <uid> --page 1 --page-size 20
```

## 发送规则

- 发消息前先确认目标对象已经是好友，必要时先查好友状态。
- 首条消息尽量具体，说明共同点、合作背景或来意。
- 不要代用户发送外部联系方式，除非用户明确要求。

## 历史记录

- `chat history` 用于回顾上下文、总结对话进展或决定下一步回复。
- 分页读取即可，不要默认拉取很长历史。

## 常见错误

- `40004`：当前对象不可聊天，通常需要先建立好友关系。
- `40101` / `40102`：先回到认证流程。

## 结果整理建议

- 返回聊天摘要时，按时间整理双方关键结论和待办。
- 如果只是给用户看近期消息，优先展示最近几条，不要倾倒完整历史。
